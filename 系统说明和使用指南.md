# 旅游攻略系统 - 完整功能说明

## 📋 系统概述

本系统是一个完整的旅游攻略生成和管理平台,支持:
- ✅ 全国50+主要城市的完整旅游数据
- ✅ 实时天气数据同步(集成和风天气API)
- ✅ 城市热门度实时更新
- ✅ 手动/自动数据同步
- ✅ 城市管理面板

## 🚀 快速开始

### 1. 启动后端服务

```bash
cd backend
npm start
```

服务将运行在 `http://localhost:3001`

### 2. 打开前端页面

- **主页面**: `index.html` - 旅游攻略生成器
- **管理面板**: `admin.html` - 城市管理后台

## 📊 已实现功能

### 一、后端API服务

#### 1. 城市数据接口

| 接口 | 方法 | 描述 |
|------|------|------|
| `GET /api/cities` | GET | 获取所有城市列表 |
| `GET /api/cities/all` | GET | 获取所有城市详细数据(含实时信息) |
| `GET /api/cities/:name` | GET | 获取指定城市详情 |
| `POST /api/cities` | POST | 添加新城市 |
| `PUT /api/cities/:name` | PUT | 更新城市数据 |
| `DELETE /api/cities/:name` | DELETE | 删除城市 |

#### 2. 实时数据接口

| 接口 | 方法 | 描述 |
|------|------|------|
| `GET /api/weather/:name` | GET | 获取城市实时天气 |
| `GET /api/trending` | GET | 获取热门城市排行榜 |
| `POST /api/sync-weather/:name?` | POST | 手动同步天气数据 |
| `POST /api/sync/:name` | POST | 同步单个城市数据 |
| `POST /api/sync-all` | POST | 同步所有城市数据 |

#### 3. 统计信息

| 接口 | 方法 | 描述 |
|------|------|------|
| `GET /api/stats` | GET | 获取系统统计信息 |

## 🔧 技术架构

### 后端

- **框架**: Node.js + Express
- **数据存储**: MongoDB / 内存存储
- **天气API**: 和风天气(可选)
- **定时同步**: 每10分钟自动更新
- **缓存机制**: 智能缓存,减少API调用

### 前端

- **原生JavaScript**: 无依赖,轻量快速
- **实时同步**: 每5分钟自动刷新
- **响应式设计**: 支持各种设备
- **本地存储**: localStorage缓存数据

### 数据流程

```
[用户访问] → [前端index.html]
              ↓
        [请求后端API]
              ↓
    [后端server.js:3001]
              ↓
    [城市数据 + 实时天气]
              ↓
        [返回前端显示]
              ↓
    [定时刷新(5分钟)]
```

## 🌐 和风天气API配置

1. 访问 [https://dev.qweather.com/](https://dev.qweather.com/) 注册账号
2. 创建应用并获取API Key
3. 在 `backend/.env` 文件中配置:

```env
QWEATHER_API_KEY=your-api-key-here
```

如果不配置,系统将使用模拟天气数据。

## 🐛 常见问题

### Q: 后端服务无法启动?

A: 检查Node.js是否安装,端口3001是否被占用。

### Q: 天气数据无法获取?

A: 检查和风天气API Key是否正确,网络是否通畅。未配置时使用模拟数据。

### Q: 前端无法连接后端?

A: 确保后端服务已启动,访问`http://localhost:3001/api/cities`是否正常。

### Q: 如何添加新城市?

A: 使用城市管理面板(admin.html)或通过API POST请求添加。

## 🚀 性能优化

1. **批量加载**: 一次性获取所有城市数据,减少请求次数
2. **智能缓存**: 天气数据缓存10分钟,避免频繁请求
3. **按需加载**: 只加载当前显示城市的详细信息
4. **本地存储**: 使用localStorage缓存常用数据
5. **并行请求**: 同时请求多个城市数据,提高加载速度
